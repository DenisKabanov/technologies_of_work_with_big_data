Запуск контейнеров:
docker-compose up
С указанием специального файла (отличного от docker-compose.yml):
docker-compose -f <YML_FILE_NAME> up
Например:
docker-compose -f docker-compose.yml up

Вывод запущенных процессов:
docker ps
Будет что-то типо:

Отправка файла в HDFS"
docker cp <PATH_TO_FILE> <CONTAINER NAME>:<DIR_TO_SAVE>
Например:
docker cp ./data/data_simple.csv namenode:/

Просмотр томов:
docker volume inspect <VOLUME_NAME>
Например:
docker volume inspect hadoop

Подключение к запущенному докер-контейнеру:
docker exec -it namenode bash

    Создание папок в HDFS:
    hdfs dfs -mkdir /data/

    Отправка данных в HDFS (что уже лежат в контейнере):
    hdfs dfs -put <FILE_TO_TRANSFER> <HDFS_STORE_DIR>
    Например:
    hdfs dfs -put /data_simple.csv /data/
    Или, с указанием размера блока:
    hdfs dfs -D dfs.block.size=32M -put /data_simple.csv /data/

    Выставление ограничение на используемую память:
    hdfs dfsadmin -setSpaceQuota <MEMORY> <PATH>
    Например:
    hdfs dfsadmin -setSpaceQuota 5g /

    Вывод основных статистик/параметров HDFS:
    hdfs dfsadmin -report

    Вывод значения параметра safemode в HDFS:
    hdfs dfsadmin -safemode get
    Вывод:
    Safe mode is ON или OFF

    Изменение safemode в HDFS на OFF:
    hdfs dfsadmin -safemode leave

Копирование данных из контейнера на host устройство:
docker cp <CONTAINER_NAME>:<PATH_TO_FILE_ON_CONTAINER> <PATH_TO_FILE_ON_HOST>
Например:
docker cp spark-master:/log.txt ./logs/log_DataNodes_3_opt_True.txt

Выход из контейнера (или любого вложенного терминала):
exit

Отключение docker-compose (всех указанных в файле контейнеров):
docker-compose down
